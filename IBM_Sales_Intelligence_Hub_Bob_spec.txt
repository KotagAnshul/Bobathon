Your task is to create a Sales Intelligence Hub web dashboard prototype powered by IBM Bob (watsonx Orchestrate).

This is a demo prototype for IBM Sales Representatives. Since this is a prototype, you will hardcode mock data
throughout the application. Mock data should reflect a realistic enterprise account (use "Accenture" as the
sample account) and realistic IBM ISC Cloud data. The dashboard should use IBM's official color palette
(IBM Blue #0F62FE, Carbon Design System neutrals) and follow a professional, enterprise-grade layout.

When the user logs in, they select the account they are working with as a Sales Representative. All sections
of the dashboard then populate with data scoped to that chosen account.

---

DASHBOARD SECTION OVERVIEW:

The dashboard is organized into the following tabs/sections:
  1. Morning News & Alerts
  2. Account Overview
  3. Competitor Analysis
  4. Executive Summary
  5. Meetings & Relationships (Outlook / Teams Integration)
  6. Current IBM Partnerships & Relationships
  7. AI Agent Launchpad (Pre-built IBM Sales Agents)
  8. AI Chatbot (Ask Bob / AskSales)

---

AGENT IMPLEMENTATION DETAILS:

1. Agent instructions are implemented in YAML. Use the YAML guidelines on this webpage:
   https://developer.watson-orchestrate.ibm.com/agents/build_agent

2. You will create the following agents:
     - morning_news_agent
     - account_intelligence_agent
     - competitor_analysis_agent
     - meetings_relationships_agent
     - ai_agent_launchpad_agent
     - sales_hub_orchestrator_agent (the orchestrator that routes to all sub-agents above)

3. Implementation of agents:

   1. morning_news_agent: Responsible for surfacing live and recent news relevant to the chosen account.
      In production, this agent will use Tavily to scrape the internet for press releases, public filings,
      product announcements, executive changes, and competitor news on a daily refresh cycle.
      For this prototype, return hardcoded mock news items.
      This agent has the following tools:
        a. get_account_news(account_name: str): Returns 4-6 mock recent news items (title, source, date, summary,
           and a "relevance tag" such as "Opportunity", "Risk", "Competitor Move", or "Executive Change") for
           the chosen account.
        b. get_industry_alerts(industry: str): Returns 2-4 mock industry-level alerts relevant to the account's
           sector (e.g., "Financial Services AI Regulation Update", "Cloud Spending Trends Q2 2026").
        c. flag_news_item(news_id: str, flag_type: str): Allows a sales rep to flag a news item as
           "Important", "Share with Team", or "Dismiss". Returns a confirmation message.

   2. account_intelligence_agent: Responsible for surfacing structured account details and ISC-connected data.
      This agent has the following tools:
        a. get_account_details(account_name: str): Returns mock account profile data including company
           overview, industry, HQ location, revenue, employee count, IBM account tier, primary IBM contact,
           and current stage in the sales cycle.
        b. get_isc_cloud_data(account_name: str): Returns mock IBM ISC Cloud data including active IBM
           products/licenses, renewal dates, usage metrics, and open support tickets.
        c. get_executive_summary(account_name: str): Returns a mock executive summary with 4-6 bullet points
           tailored for a sales rep preparing for a client call. Include key priorities, recent wins,
           open opportunities, and recommended next steps.

   3. competitor_analysis_agent: Responsible for delivering competitive intelligence for the chosen account.
      This agent has the following tools:
        a. get_competitors(account_name: str): Returns a list of 3-5 competitors currently engaged with or
           being evaluated by the account, including competitor name, products being considered, estimated
           deal size, and IBM's competitive position (e.g., "Advantaged", "Even", "At Risk").
        b. get_competitor_battlecard(competitor_name: str): Returns a mock competitive battlecard with
           IBM's differentiators, key weaknesses of the competitor, suggested talking points, and relevant
           IBM products to position.
        c. get_win_loss_summary(account_name: str): Returns a mock summary of recent IBM wins and losses
           at this account, with brief reasoning.

   4. meetings_relationships_agent: Responsible for surfacing meeting history, upcoming meetings, and
      relationship data sourced from Outlook and Microsoft Teams.
      In production this agent connects to the Microsoft Graph API.
      For this prototype, return hardcoded mock data.
      This agent has the following tools:
        a. get_upcoming_meetings(account_name: str): Returns 2-4 mock upcoming meetings with the account,
           including meeting title, date/time, attendees (IBM side and client side), and a preparation
           status flag ("Prep Complete", "Needs Prep", "No Brief").
        b. get_meeting_history(account_name: str): Returns the last 5 meetings held with the account,
           including date, attendees, topic, and a one-sentence outcome summary.
        c. get_relationship_map(account_name: str): Returns a mock relationship map showing key client
           contacts, their seniority, IBM relationship owner, engagement level
           ("Strong", "Moderate", "Cold"), and last interaction date.
        d. log_meeting_note(meeting_id: str, note: str): Accepts a note from the sales rep and returns
           a confirmation that the note was saved to the CRM.

   5. ai_agent_launchpad_agent: Responsible for launching and coordinating the pre-built IBM Sales AI Agents.
      Each sub-function routes the user to the relevant IBM agent or surfaces results from it.
      This agent has the following tools:
        a. run_lead_identification(account_name: str): Returns 3-5 mock identified leads within the account,
           including contact name, title, department, and a lead score (1-100).
        b. run_client_outreach_draft(contact_name: str, context: str): Returns a mock personalized
           email draft for the specified contact, incorporating the provided context. Simulates the
           IBM Client Outreach AI Agent powered by Salesforce and Seismic content.
        c. run_client_search(account_name: str): Returns enriched company and contact data from a mock
           Dun & Bradstreet source, including firmographic data, technographics, and key contacts.
        d. get_crm_snapshot(account_name: str): Returns a mock Salesforce CRM snapshot including open
           opportunities, pipeline stage, last activity date, and forecast category.
        e. update_crm_record(record_id: str, field: str, value: str): Returns a confirmation that the
           specified CRM field was updated in Salesforce.
        f. get_enablement_content(topic: str): Returns 2-3 mock Seismic content recommendations
           (title, content type, relevance score) for the given sales topic or product.
        g. run_sales_research(account_name: str): Returns a consolidated mock research brief combining
           Dun & Bradstreet data, news signals, and IBM internal data into an actionable account summary.

   6. sales_hub_orchestrator_agent: This agent is the master orchestrator. It receives natural language
      requests from the sales rep via the AI Chatbot tab and routes them to the appropriate sub-agent.
      It also powers the "Ask Bob" chatbot embedded in the dashboard. It should be capable of:
        - Answering questions about any section of the dashboard
        - Triggering tools from any sub-agent
        - Providing a unified, conversational interface that mirrors and extends the IBM AskSales bot,
          but with real-time, account-specific, automated context
        - Proactively surfacing the most important information for the rep's day
